E343 Course Alignment & Cleanup

Progress Report — March 2026

What We Inherited

The Starting Landscape

When this cleanup project began, the E343 course materials existed as a complex, distributed collection of resources spanning multiple semesters and storage systems. The core challenges were:

  • Canvas imscc Export (2.2GB): Contains duplicate content, broken internal links, and inconsistent folder organization across multiple migration cycles
  • Three Overlapping Textbooks: Van de Walle, Five Practices, and Classroom Discourse Moves each organize content differently with overlapping but distinctly-named pedagogical concepts
  • Multi-Semester Coverage: Materials from FA22, FA24, SP24, SP25, and SP26 with no clear designation of the "current" or "canonical" version
  • 100+ Illustrative Mathematics PDFs: Rich problem sets and lesson guides with inconsistent naming conventions and no clear mapping to course units
  • 26 Generic Quizzes & 13 Incomplete Assignments: Quiz titles like "Quiz 1" or "Module 2 Assessment" lack descriptive metadata; assignment rubrics and learning objectives are inconsistently documented
  • Format Fragmentation: Course materials exist in HTML, DOCX, MD, PDF, and PPTX with no unified naming or version control conventions

What's Next: Phased Plan

Implementation Roadmap

The path forward is organized into four distinct phases, each with clear deliverables and dependencies. This sequencing ensures foundation work precedes Canvas integration, which precedes content rationalization and enhancement.

1

Phase 1: Foundation (Immediate)

Goal: Clean up core technical debt and establish organizational standards

  • Fix CDM Complete Ebook duplicate identifiers and produce clean DOCX version for Canvas import
  • Archive all Math-Methods GitHub resources, then remove from public repository
  • Establish and document consistent file naming convention for all future units
  • Create unit template structure (lesson plans, presentations, quizzes, assignments)
2

Phase 2: Canvas Readiness

Goal: Prepare all content for reliable Canvas integration

  • Determine Canvas math rendering support (likely KaTeX or Canvas equation editor)
  • Build dual CSS/JS: simplified baseline for Canvas, full design for standalone hosting
  • Audit and fix all broken internal links in imscc content before re-import
  • Repackage cleaned imscc for fresh Canvas upload with corrected folder structure
  • Validate all file encoding and Canvas compatibility
3

Phase 3: Content Rationalization

Goal: Eliminate redundancy and standardize metadata

  • Use alignment graph to identify redundant content, unused resources, and orphaned materials
  • Replace page-number references with descriptive section references (e.g., "VdW Chapter 10, Division with Remainders")
  • Link Indiana Standards to live URLs rather than locally-stored PDFs
  • Standardize rubric formatting across all assignments and quizzes
  • Create unified metadata schema for all course materials
4

Phase 4: AI Enhancement & Integration

Goal: Leverage AI and external resources for deeper learning

  • Prepare comprehensive NotebookLM resource bundle (syllabus, rubrics, assignments, full schedule)
  • Implement Canvas date variables where supported for dynamic syllabus updates
  • Build integration plan for Illustrative Mathematics (pending Indiana state adoption)
  • Create student-facing vocabulary concordance across all frameworks
  • Design AI-ready assessment items for Canvas quiz engine

The Alignment Graph: A Living Map of Course Architecture

The alignment graph is the central artifact of this cleanup effort. It represents a semantic network connecting every theoretical framework, assignment, lesson plan, and learning outcome in the course. This graph serves three critical functions:

1. Discovery & Navigation: Colleagues can explore how concepts flow from Indiana Standards through Van de Walle's pedagogical frameworks, connect to Classroom Discourse Moves, and manifest in specific Canvas assignments and lesson activities.

2. Quality Assurance: The graph immediately reveals gaps (learning objectives with no assigned assessment) and redundancies (duplicate content across units or semesters).

3. Iterative Improvement: As the course evolves, the graph can be updated to reflect new materials, revised standards, or adjusted learning outcomes. It becomes a living document of pedagogical intent.
